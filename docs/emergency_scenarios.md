# 긴급(3) 시나리오 정의서

**작성일**: 2026-05-30 (D2) · **연계 문서**: [label_mapping.md](label_mapping.md)

본 문서는 4단계 라벨 중 **긴급(3)** 의 하위 카테고리·신호 패턴·반례·룰 백스톱 키워드를 정의한다. 본 정의는 다음 세 가지의 **spec**이다:

1. **합성 데이터 생성** ([scripts/synthesize.py](../scripts/synthesize.py)) — 각 카테고리의 예시 페르소나·맥락·다양성 요구사항
2. **평가 데이터 큐레이션** — 합성 hold-out 구성, 카테고리별 균형
3. **룰 기반 백스톱** — 모델 외 안전망 (AI+룰 안전망 프레이밍, [label_mapping.md §6.L1](label_mapping.md))

> ⚠️ **본 문서는 탐지(detection)를 위한 신호 정의서**다. 예시는 패턴 수준으로만 제시하며, 가해 행위 재현·세부 스크립트는 의도적으로 배제한다. 모든 예시는 "이런 신호가 보이면 긴급으로 분류한다"는 분류 목적이다.

## 0. 긴급(3)의 본질 — 왜 다른 축인가

[label_mapping.md §0](label_mapping.md) 재확인:

> `0/1/2`는 한 문장의 **혐오 강도** 축. `긴급(3)`은 **그루밍·성적 유인·협박·자해 신호 등 행위 기반 위험** 축.

이 비대칭의 실무적 함의:
- 욕설이 격하다고 긴급이 아니다 (그건 경고(2)).
- 욕설이 전혀 없어도 긴급일 수 있다 (정중한 그루밍 시도).
- **의도·맥락 신호**가 결정적 — 단어 빈도가 아니라 *패턴*을 본다.

## 1. 하위 카테고리

긴급(3)을 네 개의 하위 카테고리로 분해한다. 분류기는 단일 라벨(3)을 출력하지만, **합성·평가·룰**은 카테고리별로 설계한다.

| 코드 | 이름 | 핵심 신호 |
|---|---|---|
| 3a | **그루밍** (grooming) | 신뢰 형성 → 고립화 → 의존성 형성 → 경계 침범 패턴 |
| 3b | **성적 유인** (sexual solicitation) | 사진/만남/대화의 성적 전환 시도 |
| 3c | **구체적 협박** (concrete threats) | 시간·장소·수단이 명시된 위해 예고 |
| 3d | **자해/자살 신호** (self-harm signals) | 의도·계획·작별 인사 등 즉시 개입 필요 신호 |

## 2. 카테고리별 정의

### 3a. 그루밍

#### 정의
취약한 사용자(특히 장애·미성년·고립된 성인)에게 신뢰·의존성을 점진적으로 형성하여 추후 가해(성적 착취·금전 갈취·고립 강화)의 기반을 만드는 행위.

#### 핵심 신호 패턴
1. **비대칭 친밀감**: 처음 보는 상대에게 과도한 칭찬·관심·"너만 이해해" 식 차별화
2. **고립화 유도**: "다른 사람한테는 말하지 마", "가족/선생님은 안 도와줘"
3. **의존성 형성**: 돌봄·이동·의료·금전 도움 제안 (특히 장애 도메인)
4. **비밀 강요**: "우리 둘만의 비밀이야"
5. **경계 시험**: 사적 정보 요구, 사진 요청, 점진적 신체 접촉 언급

#### 예시 (패턴)
> "너 정말 특별해. 다른 애들은 너 같지 않아. 우리 진짜 통하는 것 같아 — 다른 사람들은 우리 이해 못 할 거야."

> "엄마한테는 말하지 말고, 그냥 나한테 다 얘기해도 돼. 내가 다 들어줄게."

> "장애 있어서 다들 무시하지? 나는 안 그래. 내가 너 챙겨줄게. 대신 나만 믿어야 해."

#### 반례 (false positive 방지)
- 친구·가족의 일상적 친밀감 표현 ("너밖에 없어 ㅋㅋ")
- 의료진·치료사·복지사의 전문적 돌봄 제공
- 일반적 격려 ("힘내", "내가 도와줄게")
→ **단일 발화는 그루밍이 아니다**. 패턴(고립화+의존성+비밀)이 결합되어야 함.

#### 룰 백스톱 (보조 신호, 단독 결정 X)
```
- 비밀 강요: "둘만의 비밀", "말하지 마", "아무한테도"
- 고립화: "가족/엄마/아빠/선생님이 (안 도와줘|모르게|만나지 마)"
- 의존성 유도: "내가 다 해줄게", "너 (장애|혼자)니까 내가"
- 비대칭 친밀감: 첫 대화 N턴 내 "특별하다|너밖에"
```

### 3b. 성적 유인

#### 정의
대화·만남·이미지 교환을 성적 목적으로 전환하려는 시도. 직접적 성적 요구뿐 아니라 점진적 유도도 포함.

#### 핵심 신호 패턴
1. **사진 요청**: 신체 사진·"셀카"·"부위" 언급
2. **만남 유도 + 사적 공간**: 호텔·집·인적 드문 장소
3. **대화 주제의 성적 전환**: 일상 → 외모 → 신체 → 성적 경험
4. **금전 + 만남**: "용돈 줄게", "스폰" — 특히 미성년·취약 사용자에게
5. **수위 시험**: "야한 얘기 해본 적 있어?", "이런 거 본 적 있어?"

#### 예시 (패턴)
> "사진 한 장만 보내줘. 너 진짜 예쁠 것 같아서."

> "오늘 저녁에 우리 집에서 영화 볼래? 아무도 없어."

> "성인이지? 야한 거 좋아해?"

#### 반례
- 의학·교육·예술 맥락의 신체 언급
- 합의된 성인 간 명시적 성적 대화 (다만 본 분류기는 안전 도구이므로 보수적으로 판정)
→ 1:1 채팅·미성년·취약 사용자 대상이면 임계값 낮춤.

#### 룰 백스톱
```
- 사진 요청 + 신체: "(사진|셀카|얼굴|몸|부위).*보내"
- 사적 공간 만남: "(우리|내) (집|방|호텔).*(올래|오면|봐|만나)"
- 수위 떠보기: "(야한|성|섹스).*(해봤|좋아|관심)"
- 금전 미끼: "(용돈|스폰|돈).*(줄게|줄 수)"
```

### 3c. 구체적 협박

#### 정의
타인 또는 자신에 대한 위해를 **시간·장소·수단 중 하나 이상이 구체화된 형태**로 예고. 막연한 분노 표현·욕설은 경고(2), 구체화되면 긴급(3).

#### 핵심 신호 패턴
1. **시간 명시**: "오늘 밤", "내일 X시", "퇴근하고"
2. **장소 명시**: 학교·집·직장 주소 또는 위치
3. **수단 명시**: 칼·총·차량·구체적 도구
4. **타겟 특정**: 이름·직책·관계
5. **공개 가능 위협**: 사진·영상·개인정보 유포 예고 (디지털 협박)

#### 예시 (패턴)
> "오늘 밤에 너희 학교 앞에서 기다린다. 두고 봐."

> "주소 다 알아. 내일 찾아간다."

> "이 사진 SNS에 다 뿌릴 거야. 30분 안에 답해."

#### 반례
- 비유적 분노 ("죽여버리고 싶다" — 시간·장소·수단 없음 → 경고(2))
- 농담·게임 컨텍스트 ("게임에서 너 죽인다 ㅋㅋ")
- 가정·소설·창작 맥락
→ **구체성**이 결정 기준. 막연한 분노는 경고(2)로 처리.

#### 룰 백스톱
```
- 시간 + 위해: "(오늘|내일|밤|새벽|이따).*(죽|패|찌|보내|잡)"
- 장소 + 위해: "(집|학교|회사|주소).*(찾|간|기다|보내)"
- 디지털 협박: "(사진|영상|녹음).*(올린|뿌린|유포|공개)"
- 카운트다운: "(\d+)\s*(분|시간|초)\s*(안|내|만)"
```

### 3d. 자해/자살 신호

#### 정의
사용자 본인의 자해·자살을 **의도·계획·작별 형태**로 표현. 일상적 비유("죽겠다 ㅋㅋ")와 구분 필요.

#### 핵심 신호 패턴
1. **의도 표명**: "끝낼래", "더 못 버틸 것 같아", "사라지고 싶어"
2. **계획·수단 언급**: 특정 방법·장소·시기 (구체적이면 즉시 개입)
3. **작별 인사**: "그동안 고마웠어", "다들 잘 살아"
4. **수단 확보 정황**: "약 모았어", "옥상 갔다 왔어"
5. **이전 시도 언급 + 현재 의도**

#### 예시 (패턴)
> "이번에는 진짜로 할 거야. 다들 미안해."

> "약 다 모았어. 오늘 밤이면 끝나."

> "이 메시지가 마지막일 거야."

#### 반례
- 일상적 과장 ("아 진짜 죽겠네 시험 망함 ㅠㅠ")
- 영화·드라마·게임 언급
- 추상적 우울감 ("힘들어", "지친다") — 주의(1)~경고(2) 수준, 긴급은 의도/계획이 결합
→ **다만 본 도메인은 안전 도구**이므로 임계값은 보수적. 추상 우울감도 별도 케어 트랙(주의/경고)으로 처리 가능.

#### 룰 백스톱
```
- 의도 + 종결: "(끝낼|끝내|사라질|없어질|이번이 마지막)"
- 수단 명시: "(약|밧줄|뛰어|옥상|다리).*(모았|준비|갔다)"
- 작별: "(그동안|마지막).*(고마|미안|잘 살)"
- 부정적 시간감각: "더 (못|안) (버|살)"
```

> 🆘 **참고**: 룰 백스톱이 트리거되면 모델 출력과 별개로 **즉시 위기 대응 자원 안내** (예: 자살예방상담전화 109)을 함께 노출. 본 문서 범위 밖이지만 제품 정책에 명시 권고.

## 3. 경계 케이스 (Disambiguation)

긴급(3) 판정이 까다로운 케이스 — 합성·평가 데이터에 의도적으로 포함하여 모델을 단련한다.

| 케이스 | 후보 라벨 | 결정 기준 |
|---|---|---|
| "죽고 싶다" 단발 (맥락 없음) | 주의(1) / 경고(2) | 다른 자해 신호 없으면 일상어로 판정, 주의 |
| 의료진의 정중한 친밀감 | 정상(0) | 직업·맥락 명시되면 정상 |
| 게임 채팅의 "죽인다" | 정상(0) ~ 주의(1) | 게임 컨텍스트 명시되면 정상~주의 |
| 막연한 분노 ("패고 싶다") | 경고(2) | 구체성 없으면 경고 |
| 정중한 그루밍 (욕설 0) | 긴급(3a) | 패턴(고립화+의존성+비밀)이면 긴급 |
| 합의 성인 간 성적 대화 | 정상(0) | 본 도구는 1:1 안전 감시 컨텍스트라 임계값 낮음 — 단, 미성년·취약 신호 결합되면 긴급 |
| "마지막 메시지야 잘 자" | 정상(0) | 단순 인사면 정상, 작별 + 부정 시간감각이면 긴급 |

## 4. 장애 도메인 특이성

본 프로젝트는 장애인 커뮤니티가 주 도메인이다. 일반 그루밍·협박 패턴과 다른 신호:

### 4a. 의존성 악용
- **돌봄 미끼**: "내가 활동지원 해줄게", "병원 같이 가줄게"
- **이동 미끼**: 휠체어·장애인콜택시·이동 도움 제안 — 정상적 도움 vs 그루밍 패턴 구분
- **의료 정보 악용**: 복용약·진료 정보 캐묻기

### 4b. 차별·고립 강화 발화 (경고와 긴급의 경계)
- 일반 혐오: "장애인 주제에" → 경고(2)
- 자해 유도: "장애 있으면 부담만 되니까 사라지는 게 나아" → **긴급(3d 변형)** — 타자의 자해 권유는 긴급 처리
- 보호자 분리 시도: "엄마는 너 짐스러워해" → **긴급(3a 변형)** — 고립화

### 4c. 발달장애·인지장애 특이 신호
- 친밀감의 일방성·과속도 (수신자가 자각·거부 어려움)
- 보상·선물·게임 아이템 미끼
- "비밀" 강요의 효력이 강해 더 위험

### 합성 요구사항
- **전 카테고리(3a~3d)에 장애 페르소나·맥락 비중 ≥ 30%**
- 보호자·활동지원사·치료사 등 신뢰 관계 인물 사칭 시나리오 포함
- 의료·돌봄 어휘 자연스러움 (어색하면 일반 혐오 데이터 노이즈로 분리됨)

## 5. 합성 데이터 요구사항 (D2 입력)

### 5.1 카테고리별 목표 합성량
| 카테고리 | train | val | test | 메모 |
|---|---:|---:|---:|---|
| 3a 그루밍 | 200 | 30 | 50 | 패턴 다양성 핵심 |
| 3b 성적 유인 | 150 | 25 | 40 | 직접·점진적 둘 다 |
| 3c 구체적 협박 | 150 | 25 | 40 | 시간/장소/수단 조합 |
| 3d 자해/자살 신호 | 100 | 20 | 30 | 보수적 (윤리 검토) |
| **소계** | **600** | **100** | **160** | 약 860건 |
| **반례 (boundary)** | 200 | 50 | 60 | 정상·주의·경고 흡사 케이스 |
| **총합** | **800** | **150** | **220** | 약 1,170건 |

**비용 추정**: GPT-4o-mini 기준 ≈ $3~5 (1.17K 샘플, 토큰당 비용 고려).

### 5.2 다양성 축 (페르소나 × 컨텍스트)
- **화자 페르소나**: 또래·연상·익명·"도와주는 사람"·"의존 관계"
- **수신자 페르소나**: 일반 사용자, 장애 사용자 (전 카테고리 ≥30%)
- **컨텍스트**: 1:1 채팅, 커뮤니티 댓글, DM 첫 메시지, N턴 뒤 발화
- **언어 스타일**: 정중함 ↔ 거친 욕설, 표준어 ↔ 사투리·이모티콘

### 5.3 합성 프롬프트 설계 원칙
1. **카테고리 정의 + 패턴 + 반례를 모두 프롬프트에 포함** — 단순히 "그루밍 예시 만들어"가 아니라 본 문서의 핵심 신호 패턴을 명시
2. **장애 도메인 페르소나 사전 정의** — 발달장애·시각장애·청각장애·지체장애·정신장애별 자연스러운 어휘
3. **반례 생성 별도** — 정상/주의/경고로 분류돼야 할 경계 케이스를 의도적으로 생성하여 라벨 노이즈 방지
4. **윤리 가드레일**: 자해 카테고리 생성 시 OpenAI 안전 정책 준수, 실제 가해 스크립트 생성 회피

### 5.4 합성 데이터 저장 규약
- `data/synthetic/emergency/{3a,3b,3c,3d,boundary}/{train,val,test}.jsonl`
- 각 행: `{"text": ..., "label": 3, "subcategory": "3a", "persona": ..., "context": ..., "generated_at": ...}`
- 라벨은 모두 3 (boundary는 0/1/2 중 하나)
- **반드시** `source: "synthetic_emergency_v1"` 필드로 시드와 분리 추적

## 6. 평가 데이터 설계 — L1 완화

[label_mapping.md §6.L1](label_mapping.md) 의 측정 불가 문제 완화 방안.

### 6.1 평가 셋 분리
| 평가 셋 | 출처 | 용도 |
|---|---|---|
| **Seed Hold-out** | data/processed/test.parquet | 모듈 ① 기본 성능 (긴급 제외 3-class 의미만) |
| **Synthetic Emergency Hold-out** | data/synthetic/emergency/test/ | 긴급 클래스 성능 — *합성 기준* 명시 |
| **Curated Hard Cases** | 수동 작성 ~50건 | 경계 케이스 별도 검증 (시간 되면) |

### 6.2 리포트 시 의무 표기
- 통합 Macro-F1과 **데이터셋별 F1 분리 보고**
- 긴급 F1은 **항상** "(synthetic eval, n=X)" 표기
- 룰 백스톱과 모델의 결합 metric: precision/recall@operating threshold

### 6.3 룰 + 모델 결합 평가
- 모델 단독 vs (모델 OR 룰) vs (모델 AND 룰) 세 가지 측정
- 안전 도구 특성상 **Recall 가중치 높음** (false negative 비용 > false positive)

## 7. 룰 백스톱 — 통합 정책

각 카테고리 §2의 정규식·키워드를 통합하여 `src/rules/emergency_rules.py` (D2 후반 또는 D3 작성)에 구현.

### 운영 원칙
1. **OR 결합**: 모델 예측 3 ∨ 룰 매치 → 긴급 처리 (Recall 우선)
2. **임계값 조정 가능**: 1:1 채팅·취약 사용자에서는 엄격, 게임 채팅에서는 완화
3. **룰 매치 시 응답 정책**: 자해(3d)는 위기 자원 안내 동시 노출, 협박(3c)은 신고 안내, 그루밍/성적유인은 차단 + 보호자 알림 옵션 (제품 정책)
4. **로그·분석**: 룰 매치 빈도 별도 로깅 → 정밀도 모니터링

## 8. 한계·후속 작업

### 본 정의서의 한계
1. **실제 데이터로 검증 안 됨** — 합성 기반 정의이며, 실제 그루밍·협박 발화 분포와 다를 수 있음
2. **언어·문화 의존성** — 한국어·한국 문화권 패턴 중심, 외국어 혼용·번역 공격 대응 미흡
3. **다중 턴 맥락 불완전** — 단일 메시지 기반 정의가 많아 그루밍(N턴 누적)은 부분적 탐지만 가능
4. **윤리 검토 부재** — 본 학기 범위에서 IRB·전문가 자문 없음. 후속 연구 필수

### 후속 작업 (학기 외)
- 실제 사례 (익명화·동의 기반) 수집·라벨링
- 멀티턴 모델로 확장 (대화 단위 분류)
- 도메인 전문가 (장애인권 단체·아동보호·정신건강) 검토
- 다국어 확장
