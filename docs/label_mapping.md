# 4단계 위험도 라벨 매핑 정책

**작성일**: 2026-05-30 (D1) · **근거 EDA**: [notebooks/01_eda.ipynb](../notebooks/01_eda.ipynb)

본 문서는 시드 데이터셋(Smilegate UnSmile, KOLD)을 본 프로젝트의 **4단계 위험도 라벨**(`정상/주의/경고/긴급`)로 매핑하는 규칙과 그 근거를 명시한다. 결정 사항은 굵게 표시하고, 알려진 한계는 마지막 섹션에 분리한다.

---

## 0. 가장 먼저: 긴급(3)은 다른 축이다

`정상(0) / 주의(1) / 경고(2)`는 **한 문장 내 혐오 강도** 축이다.
`긴급(3)`은 **그것의 강한 버전이 아니다**. 다음과 같이 정의한다:

> **긴급(3) = "한 문장의 강한 혐오"가 아니라, 그루밍·성적 유인·구체적 협박·자해/자살 신호 등 행위 기반 위험.**
> 일반적으로 멀티턴 맥락에서 드러나며, 단발 욕설의 강도와는 다른 축이다.

**왜 시드에 0건인가**: 데이터가 부족해서가 아니라, UnSmile/KOLD가 **단발 혐오 댓글 데이터셋**이라 이 축을 애초에 담고 있지 않다. 따라서:

- 긴급(3)은 **시드 매핑으로는 얻을 수 없음**.
- D2 합성도 "더 심한 욕설"이 아니라 **"그루밍·유인·협박 시나리오"** 로 정의해야 한다.
- 긴급(3) 측정의 한계는 본 문서 §6 "알려진 제약"에 별도 명시.

## 1. 4단계 라벨 공식 정의

| 코드 | 라벨 | 정의 | 축 |
|---|---|---|---|
| 0 | 정상 | 위험 신호 없음. 일상 대화·일반 의견. | 강도 |
| 1 | 주의 | 부적절·무례한 표현. 욕설·짜증·비꼼. 보호집단 차별성은 미약. | 강도 |
| 2 | 경고 | 명시적 보호집단 차별·혐오·괴롭힘. 자기검열 권고 수준. | 강도 |
| 3 | 긴급 | 그루밍, 성적 유인, 구체적 협박, 자해/자살 신호 등 **행위 기반 위험**. | **행위** |

`0~2`는 같은 척도 위의 단계, `3`은 다른 척도 위에 있다는 점이 본 프로젝트 라벨 체계의 핵심 비대칭이다.

## 2. UnSmile 매핑 규칙

UnSmile은 **11개 멀티라벨** (그룹 혐오 7 + 기타 혐오 + 악플/욕설 + clean + 개인지칭). 매핑 규칙은 다음 우선순위로 적용 (위에서 아래로):

```
1) clean == 1                                            → 0 (정상)
2) 그룹 혐오 라벨 ≥ 1개                                   → 2 (경고)
   (여성/가족, 남성, 성소수자, 인종/국적, 연령, 지역, 종교, 기타 혐오)
3) 악플/욕설 == 1 AND 그룹 혐오 == 0                       → 1 (주의)
4) 그 외                                                  → 0 (fallback, 거의 발생 안 함)
```

- **멀티라벨 collapse 정책**: 한 행에 그룹 혐오 + 악플/욕설이 동시에 켜져 있으면 **최고 등급(경고)** 으로 흡수. 위 규칙 2가 3보다 먼저 적용되는 것으로 자동 처리됨.
  - 위험 라벨 2개 켜진 행: 905건 / 3+개: 89건 — 모두 경고(2)로 흡수
- `개인지칭` 라벨은 보조 신호 — 본 collapse에서는 사용하지 않음. *§5에서 후처리 룰의 후보 신호로 재등장.*

### 예상 분포 (train+valid, n=18,742)
| 라벨 | 개수 | 비율 |
|---|---:|---:|
| 정상 | 4,676 | 25.0% |
| 주의 | 3,929 | 21.0% |
| 경고 | 10,137 | 54.1% |
| 긴급 | 0 | 0% |

## 3. KOLD 매핑 규칙

```
1) OFF == False                                           → 0 (정상)
2) OFF == True AND TGT == "group"                          → 2 (경고)
3) OFF == True AND TGT == "individual"                     → 1 (주의)  ※ §5 참조
4) OFF == True AND TGT ∈ {"untargeted", "other"}           → 1 (주의)
```

### 결정 사항 — TGT=other (n=1,402)
- 1,402건 모두 OFF=True, GRP=None.
- **2026-05-30 샘플링 검토 (40건)**: 대부분 언론·정부·기관·정책에 대한 분노·조롱·짜증. 보호집단 직접 차별은 0~2건 (≤5%).
- **결정**: **주의(1)로 흡수**. 경고급 노이즈 ≤ 5%로 추정되어 라벨 노이즈 수용 가능.

### 예상 분포 (n=40,429)
| 라벨 | 개수 | 비율 |
|---|---:|---:|
| 정상 | 20,119 | 49.8% |
| 주의 | 7,897 | 19.5% |
| 경고 | 12,413 | 30.7% |
| 긴급 | 0 | 0% |

## 4. 두 데이터셋 통합 — 정의 차이 명시

**중요**: 두 데이터셋은 같은 4단계 라벨에 매핑되지만, **경고(2)의 정의가 데이터셋마다 다르다**.

| 데이터셋 | 경고(2)의 정의 |
|---|---|
| UnSmile | 보호집단 혐오 라벨이 **존재** (여성/가족, 남성, 성소수자, 인종/국적, 연령, 지역, 종교, 기타 혐오 중 ≥1) |
| KOLD | OFF=True 이면서 **타겟 유형이 group** |

→ 같은 "경고"여도 UnSmile은 "혐오 라벨 존재 여부", KOLD는 "표적 유형"에서 판정한다. **경계가 데이터셋마다 다를 수 있음**을 인정한다.

**통합 정책**: 두 데이터셋을 단순 concat하고 위 규칙으로 각자 매핑한 뒤 한 학습셋으로 합친다. 모델은 두 정의의 합집합을 "경고"로 학습한다.

**의도된 결과**: 경고 클래스 내부에 정의 이질성이 존재 — 일종의 자연적 데이터 증강. 다만 평가 시 데이터셋별 성능을 분리해서 봐야 한다 (§6 알려진 제약).

### 통합 시드 분포 (n=59,171)
| 라벨 | UnSmile | KOLD | 합계 | 비율 |
|---|---:|---:|---:|---:|
| 정상 | 4,676 | 20,119 | **24,795** | 41.9% |
| 주의 | 3,929 | 7,897 | **11,826** | 20.0% |
| 경고 | 10,137 | 12,413 | **22,550** | 38.1% |
| 긴급 | 0 | 0 | **0** | 0.0% |

## 5. 학습 라벨 ≠ 제품 정책 — 후처리 분리

### 결정 — KOLD TGT=individual 처리
- **학습 라벨**: 주의(1) **유지**. 모델 자체는 "한 문장의 혐오 강도" 축에 충실.
- **제품 정책**: 시나리오 B(1:1 채팅)에서는 후처리 룰로 **개인 표적 공격을 경고(2)로 격상**.
  - 학습된 모델이 주의(1)로 분류했더라도, 1:1 컨텍스트에서 수신자가 직접 표적인 경우(`TGT=individual` 신호) 제품 레이어에서 경고로 노출.
  - UnSmile의 `개인지칭=1` 라벨도 동일한 후처리 신호 후보.

### 근거
1. **모델은 단순하게**: 다른 한국어 혐오 분류 연구·데이터셋과 일관된 "혐오 강도" 축을 유지하면 해석성·재현성·외부 검증이 깔끔하다.
2. **제품 정책은 컨텍스트 의존**: 1:1 채팅에서 취약 사용자가 직접 표적이 되는 신호는 피드의 집단 혐오보다 더 급할 수 있다. 이는 모델이 아니라 **제품 안전 정책** 수준에서 결정해야 한다.
3. **분리의 이점**: 모델 재학습 없이 제품 정책을 바꿀 수 있다. 1:1·그룹·공개 피드 등 컨텍스트별로 다른 임계값·격상 룰 적용 가능.

## 6. 알려진 제약 (Known Limitations)

본 매핑 정책으로부터 직접 따라 나오는 **본 프로젝트가 측정·주장할 수 없는 것들**.

### L1. 긴급(3) 성능을 시드만으로는 측정 불가
- 시드 0건 → train·eval **둘 다 합성**.
- 합성으로 학습하고 합성으로 평가하면 모델이 실제 그루밍·유인을 얼마나 잡는지 **숫자로 증명할 수 없다**.
- **대응**:
  - (a) 합성을 다양하게 (페르소나·시나리오·언어 스타일 다양화), 외부 키워드/룰 백스톱 병행 → "AI 단독"이 아니라 **"AI + 룰 안전망"** 프레이밍.
  - (b) 리포트에서 긴급 클래스 성능은 "합성 hold-out 기준"임을 명시, 일반화 주장 자제.
  - (c) 실제 그루밍 데이터 확보는 본 학기 범위 밖 — 후속 연구 항목으로.

### L2. "심한 불균형 X" 결론은 0/1/2에 한정
- §4 통합 분포(42:20:38:0)는 0/1/2 사이에서는 가벼운 가중치로 충분.
- 긴급(3)은 합성량이 정해지기 전까지는 **극소수 클래스**. Focal Loss `alpha`만으로는 부족할 수 있음.
- **대응**: D2에서 합성 규모 결정 후 oversampling / class weight `alpha[3]` 별도 튜닝.

### L3. 장애 도메인은 전 클래스에서 희박
- '장애' 키워드 포함: UnSmile (train+valid) 86건, KOLD comment 141건, KOLD GRP 카테고리에 '장애-' **0건**.
- 본 프로젝트는 장애인 커뮤니티가 도메인이므로 시드만으로는 도메인 적응 불충분.
- **대응**: D2 합성은 긴급(3)만이 아니라 **전 클래스(0/1/2/3)의 장애 도메인 보강**으로 설계.

### L4. 경고(2) 정의 이질성
- §4에서 명시. 평가 시 **데이터셋별(UnSmile only / KOLD only) F1 분리 보고** 필요.
- 통합 학습 후 데이터셋별 성능 격차가 크면 정의 이질성이 노이즈로 작용한다는 증거.

## 7. 합성 데이터 요구사항 (D2 인입)

본 매핑 정책으로부터 D2 합성이 충족해야 할 요건:

1. **긴급(3) 시나리오 정의서 작성** — 그루밍/유인/협박/자해 신호 각각의 예시·반례·경계 케이스.
2. **장애 도메인 페르소나 풀** — 전 클래스에 걸쳐 장애 관련 페르소나·상황·표현 의도적 주입.
3. **0/1/2 도메인 적응용 합성** — 시드에 부족한 장애 도메인 정상·주의·경고 케이스.
4. **합성 데이터 명확한 분리 저장** — `data/synthetic/{emergency,disability_domain}/` 로 출처 추적.
5. **eval 분리** — 합성 hold-out은 시드 hold-out과 **분리 평가**, 두 결과 병기.
